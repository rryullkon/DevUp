Chapitre 12: Tests avec Jest et Testing Library

1. Pourquoi tester vos composants React ?

Les tests assurent que vos composants fonctionnent correctement, évitent les régressions et facilitent la maintenance du code.

2. Jest

Jest est un framework de test JavaScript développé par Facebook. Il permet d’écrire des tests unitaires et intègre un moteur de simulation (mock).

Il est souvent utilisé avec React car il s’intègre parfaitement avec Babel et offre une configuration minimale.

3. React Testing Library

React Testing Library est une bibliothèque complémentaire qui facilite le test des composants React en se concentrant sur le comportement de l’utilisateur plutôt que sur l’implémentation.

4. Exemple simple de test avec Jest et Testing Library

import { render, screen, fireEvent } from '@testing-library/react';
import '@testing-library/jest-dom/extend-expect';
import React from 'react';

function Bouton({ onClick, children }) {
  return <button onClick={onClick}>{children}</button>;
}

test('le bouton affiche le texte et réagit au clic', () => {
  const handleClick = jest.fn();
  render(<Bouton onClick={handleClick}>Cliquez-moi</Bouton>);

  const bouton = screen.getByText('Cliquez-moi');
  expect(bouton).toBeInTheDocument();

  fireEvent.click(bouton);
  expect(handleClick).toHaveBeenCalledTimes(1);
});

5. Tests unitaires vs tests d'intégration

  • Tests unitaires : testent une fonction ou un composant isolé.
  • Tests d'intégration : vérifient la collaboration entre plusieurs composants.

6. Bonnes pratiques

  • Écrire des tests clairs et indépendants.
  • Tester le comportement plutôt que l’implémentation interne.
  • Utiliser des mocks pour isoler les tests.
  • Automatiser l’exécution des tests (ex : via CI/CD).
