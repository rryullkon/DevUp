Chapitre 13: Performance et optimisation

1. Pourquoi optimiser ?

Une bonne optimisation permet d’améliorer la rapidité, la fluidité et la réactivité de votre application, offrant ainsi une meilleure expérience utilisateur.

2. Utiliser React.memo

React.memo mémorise un composant fonctionnel pour éviter des rerenders inutiles lorsque ses props ne changent pas. 

import React from 'react';

const MonComposant = React.memo(function(props) {
  // rendu
});

3. Utiliser useMemo et useCallback

Ces hooks permettent de mémoriser des valeurs (useMemo) ou des fonctions (useCallback) entre les rendus, réduisant les calculs ou recréations inutiles. 

import React, { useMemo, useCallback } from 'react';

const calcul = useMemo(() => {
  // calcul coûteux
  return resultat;
}, [dep1, dep2]);

const handleClick = useCallback(() => {
  // fonction stable
}, [dep]);

4. Eviter les rendus inutiles

Optimisez la structure de vos composants et évitez de passer des props différentes inutilement (ex : objets/fonctions recréés à chaque rendu).

5. Chargement paresseux (Lazy loading)

Utilisez React.lazy et Suspense pour charger les composants uniquement lorsqu’ils sont nécessaires, réduisant ainsi la taille initiale du bundle. 

import React, { Suspense, lazy } from 'react';

const MonComposant = lazy(() => import('./MonComposant'));

function App() {
  return (
    <Suspense fallback=<div>Chargement...</div>>
      <MonComposant />
    </Suspense>
  );
}

6. Analyser et profiler

Utilisez les outils comme React Developer Tools Profiler pour détecter les points chauds de performance dans votre application.

7. Bonnes pratiques

  • Ne pas optimiser prématurément, commencez par mesurer.
  • Utiliser la décomposition en composants petits et réutilisables.
  • Minimiser les états et props pour limiter les rerenders.
  • Profiter des outils de profiling pour cibler les optimisations.
