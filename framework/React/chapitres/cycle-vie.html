Cycle de vie des composants

Comprenez le cycle de vie des composants et les méthodes associées.

Chapitre 5: Cycle de vie des composants

1. Introduction

Le cycle de vie d’un composant React correspond aux différentes phases que traverse un composant depuis sa création jusqu’à sa suppression. React fournit des méthodes pour interagir avec ces phases.

2. Les phases principales

  • Montage (Mounting) : Le composant est créé et inséré dans le DOM.
  • Mise à jour (Updating) : Le composant est mis à jour suite à un changement de props ou d’état.
  • Demontage (Unmounting) : Le composant est retiré du DOM.

3. Méthodes du cycle de vie avec les composants de classe

class Exemple extends React.Component {
  componentDidMount() {
    console.log("Composant monté !");
  }

  componentDidUpdate(prevProps, prevState) {
    console.log("Composant mis à jour !");
  }

  componentWillUnmount() {
    console.log("Composant démonté !");
  }

  render() {
    return <div>Bonjour !</div>;
  }
}

4. Hooks et cycle de vie avec les composants fonctionnels

Avec les hooks, vous pouvez gérer le cycle de vie dans les composants fonctionnels à l’aide de useEffect. 

import { useEffect } from 'react';

function Exemple() {
  useEffect(() => {
    console.log("Composant monté !");
    
    return () => {
      console.log("Composant démonté !");
    };
  }, []);

  return <div>Bonjour !</div>;
}

5. Bonnes pratiques

  • Utilisez useEffect pour gérer les effets de bord dans les composants fonctionnels.
  • Nettoyez les effets pour éviter les fuites de mémoire.
  • Préférez les composants fonctionnels modernes avec hooks plutôt que les classes sauf si nécessaire.
