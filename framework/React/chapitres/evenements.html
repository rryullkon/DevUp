Gestion des événements

Apprenez à gérer les interactions utilisateur avec les événements en React.

Chapitre 6: Gestion des événements

1. Introduction

En React, la gestion des événements se fait de manière similaire au JavaScript standard, mais avec quelques différences spécifiques à JSX.

2. Syntaxe des événements

Les événements en JSX s’écrivent en camelCase, et on leur passe une fonction comme gestionnaire. 

function MonBouton() {
  function handleClick() {
    alert('Bouton cliqué !');
  }

  return <button onClick={handleClick}>Cliquez-moi</button>;
}

3. Passer des arguments à un gestionnaire

Vous pouvez utiliser une fonction fléchée pour passer des arguments à votre gestionnaire : 

function MonBoutonAvecArgs() {
  function handleClick(message) {
    alert(message);
  }

  return (
    <button onClick={() => handleClick('Bonjour depuis React !')}>
      Clique ici
    </button>
  );
}

4. Événements courants en React

  • onClick – Clic de souris
  • onChange – Changement de valeur d’un champ
  • onSubmit – Soumission d’un formulaire
  • onMouseEnter, onMouseLeave – Survols
  • onKeyDown, onKeyUp – Touches clavier

5. Bonnes pratiques

  • Définissez vos fonctions de gestion dans le composant pour plus de lisibilité.
  • Évitez d’utiliser des fonctions anonymes dans le JSX si la performance est critique.
  • Utilisez des noms explicites pour vos gestionnaires : handleClick, handleSubmit, etc.
