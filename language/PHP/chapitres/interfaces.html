Interfaces

Définition et implémentation d'interfaces.

Chapitre 20: Interfaces

1. Qu'est-ce qu'une interface ?

Une interface définit un contrat que les classes doivent respecter. Elle déclare des méthodes sans les implémenter. Une classe qui implémente une interface s'engage à définir toutes ses méthodes.

2. Pourquoi utiliser des interfaces ?

  • Permet d’imposer certaines méthodes à une classe
  • Facilite la programmation orientée contrat
  • Utile pour le polymorphisme et l’architecture logicielle modulaire

3. Syntaxe de base

<?php
interface Affichable {
    public function afficher();
}

class Produit implements Affichable {
    public function afficher() {
        echo "Je suis un produit.<br>";
    }
}

$p = new Produit();
$p->afficher();
?>

4. Implémenter plusieurs interfaces

Une classe peut implémenter plusieurs interfaces en les séparant par une virgule. 

<?php
interface Exportable {
    public function exporter();
}

interface Imprimable {
    public function imprimer();
}

class Document implements Exportable, Imprimable {
    public function exporter() {
        echo "Document exporté.<br>";
    }

    public function imprimer() {
        echo "Document imprimé.<br>";
    }
}

$doc = new Document();
$doc->exporter();
$doc->imprimer();
?>

5. Interface + polymorphisme

Une interface permet de manipuler différents objets via un même type abstrait. 

<?php
interface Animal {
    public function parler();
}

class Chien implements Animal {
    public function parler() {
        echo "Wouf!<br>";
    }
}

class Chat implements Animal {
    public function parler() {
        echo "Miaou!<br>";
    }
}

$animaux = [new Chien(), new Chat()];
foreach ($animaux as $a) {
    $a->parler();
}
?>

6. Différence avec une classe abstraite

  • Une interface ne contient que des déclarations de méthodes (et depuis PHP 8.1, des constantes et méthodes statiques possibles)
  • Une classe abstraite peut contenir des méthodes implémentées et des propriétés
  • Une classe peut hériter d'une seule classe, mais implémenter plusieurs interfaces

7. Résumé

  • Les interfaces définissent des méthodes sans les implémenter
  • Les classes qui les implémentent doivent coder ces méthodes
  • Utile pour structurer des systèmes flexibles et cohérents
