Espaces de noms

Organisation du code avec les namespaces.

Chapitre 23: Espaces de noms

1. Qu’est-ce qu’un espace de noms ?

Un espace de noms (ou namespace) permet d’organiser le code PHP et d’éviter les conflits de noms entre classes, fonctions ou constantes. C’est particulièrement utile dans les projets de grande taille ou les bibliothèques tierces.

2. Déclaration d’un namespace

<?php
namespace MonProjet;

class Utilisateur {
    public function saluer() {
        echo "Bonjour depuis MonProjet !<br>";
    }
}
?>

Le mot-clé namespace doit être placé au tout début du fichier PHP (avant toute autre instruction, sauf declare).

3. Utiliser une classe d’un autre namespace

Pour utiliser une classe définie dans un autre espace de noms, on utilise use : 

<?php
require 'MonProjet/Utilisateur.php';

use MonProjet\Utilisateur;

$u = new Utilisateur();
$u->saluer();
?>

4. Alias de namespace

On peut créer un alias avec as pour simplifier l’utilisation : 

use MonProjet\Utilisateur as User;

$u = new User();
$u->saluer();

5. Espaces de noms imbriqués

On peut organiser le code dans des sous-espaces : 

namespace App\Controllers;

class PageController {
    public function afficher() {
        echo "Affichage d'une page.";
    }
}

6. Namespace global

Pour utiliser une classe PHP native depuis un namespace, on peut précéder son nom d’un \ : 

namespace MonProjet;

function strlen($str) {
    return "Longueur : " . \strlen($str); // Utilise la fonction globale strlen
}

7. Résumé

  • Les namespaces organisent le code et évitent les collisions de noms
  • Ils permettent une meilleure modularité et lisibilité du code
  • On utilise use pour importer des classes d’un autre namespace
  • Les namespaces doivent être déclarés au début d’un fichier
