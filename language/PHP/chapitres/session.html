Sessions et gestion des données utilisateur

Débuter avec les sessions

Qu'est-ce qu'une session ?

Une session est un mécanisme utilisé en PHP pour conserver temporairement des informations sur un utilisateur, comme son nom ou son email, pendant qu'il navigue sur un site. Ces données sont stockées côté serveur, ce qui les rend plus sécurisées que les cookies.

Création d'une session

Avant d’utiliser ou d’enregistrer des données de session, il faut démarrer la session avec la fonction session_start(). Elle doit être appelée avant tout affichage HTML.

<?php
session_start(); // Démarre la session
$_SESSION['pseudo'] = 'JohnDoe'; // Enregistre une donnée
?>

Stockage de données dans une session

Les données sont enregistrées dans la superglobale $_SESSION, un tableau associatif. Chaque valeur peut être identifiée avec une clé personnalisée.

<?php
$_SESSION['pseudo'] = 'JohnDoe';
$_SESSION['email'] = 'johndoe@example.com';
?>

Récupération de données dans une session

Pour afficher ou utiliser les données stockées, il suffit d'accéder à la superglobale $_SESSION avec les clés utilisées précédemment.

<?php
echo $_SESSION['pseudo']; // Affiche "JohnDoe"
echo $_SESSION['email'];  // Affiche "johndoe@example.com"
?>

Destruction d'une session

Pour supprimer toutes les données de session, utilisez session_destroy(). Pensez à appeler session_start() juste avant pour que PHP sache quelle session terminer.

<?php
session_start(); // Nécessaire avant de détruire
session_destroy(); // Supprime la session
?>
