Classes abstraites

Définition et utilisation des classes abstraites.

Chapitre 22: Classes abstraites

1. Qu’est-ce qu’une classe abstraite ?

Une classe abstraite est une classe qui ne peut pas être instanciée directement. Elle sert de modèle pour d'autres classes, en imposant certaines méthodes à implémenter.

2. Caractéristiques

  • Contient au moins une méthode abstraite (sans corps)
  • Peut contenir des méthodes normales (implémentées)
  • Doit être héritée et complétée par une sous-classe

3. Syntaxe d'une classe abstraite

<?php
abstract class Animal {
    abstract public function parler();

    public function respirer() {
        echo "Je respire.<br>";
    }
}

class Chien extends Animal {
    public function parler() {
        echo "Wouf !<br>";
    }
}

$rex = new Chien();
$rex->parler();    // Wouf !
$rex->respirer();  // Je respire.
?>

4. Utilité

Les classes abstraites sont utiles pour :

  • Définir une structure commune à plusieurs classes
  • Forcer certaines méthodes à être définies dans les classes enfants
  • Réduire la duplication de code avec des méthodes partagées

5. Différence avec une interface

  • Une interface ne contient que des méthodes sans corps (jusqu’à PHP 8.1)
  • Une classe abstraite peut avoir des méthodes avec ou sans implémentation
  • Une classe ne peut hériter que d'une seule classe abstraite, mais peut implémenter plusieurs interfaces

6. Exemple avec plusieurs classes

<?php
abstract class Forme {
    abstract public function aire();

    public function afficherType() {
        echo "Ceci est une forme.<br>";
    }
}

class Rectangle extends Forme {
    private $longueur;
    private $largeur;

    public function __construct($l, $L) {
        $this->longueur = $l;
        $this->largeur = $L;
    }

    public function aire() {
        return $this->longueur * $this->largeur;
    }
}

$rect = new Rectangle(4, 5);
$rect->afficherType();               // Ceci est une forme.
echo "Aire : " . $rect->aire();     // Aire : 20
?>

7. Résumé

  • On utilise le mot-clé abstract pour définir une classe ou une méthode abstraite
  • Les classes abstraites servent de base pour d'autres classes
  • Elles ne peuvent pas être instanciées directement
  • Elles aident à organiser et structurer un code orienté objet
