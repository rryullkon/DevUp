Chapitre 79: Design Patterns

1. Introduction

Les design patterns, ou modèles de conception, sont des solutions réutilisables à des problèmes courants dans le développement logiciel. En PHP, ils permettent d’écrire un code plus clair, maintenable et évolutif.

Ils sont classés en plusieurs catégories : créationnels (ex : Singleton, Factory), structurels (ex : Adapter, Decorator), comportementaux (ex : Observer, Strategy).

2. Pourquoi utiliser des design patterns ?

  • Favoriser la réutilisation du code.
  • Améliorer la lisibilité et la compréhension du code.
  • Faciliter la maintenance et l’évolution du logiciel.
  • Standardiser les solutions aux problèmes fréquents.

3. Exemple : Le pattern Singleton

Le pattern Singleton garantit qu’une classe n’a qu’une seule instance et fournit un point d’accès global à cette instance. 

<?php
class Singleton {
    private static $instance = null;

    // Constructeur privé pour empêcher l'instanciation directe
    private function __construct() {}

    // Empêcher la duplication de l'objet
    private function __clone() {}

    // Méthode statique pour récupérer l'instance unique
    public static function getInstance() {
        if (self::$instance === null) {
            self::$instance = new Singleton();
        }
        return self::$instance;
    }

    public function doSomething() {
        return "Action effectuée par l’instance unique.";
    }
}

// Utilisation
$singleton = Singleton::getInstance();
echo $singleton->doSomething();
?>

4. Autres design patterns courants

  • Factory : création d’objets sans exposer la logique de création.
  • Observer : permet de notifier automatiquement plusieurs objets en cas de changement.
  • Decorator : ajoute dynamiquement des fonctionnalités à un objet.
  • Strategy : permet de changer l’algorithme utilisé à la volée.
