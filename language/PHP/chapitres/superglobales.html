Superglobales

$_GET, $_POST, $_SERVER, $_SESSION, etc.

Chapitre 32: Superglobales

1. Qu'est-ce que les superglobales ?

Les superglobales sont des variables prédéfinies en PHP accessibles partout dans le script, même à l'intérieur des fonctions sans avoir besoin de les passer en paramètre.

2. $_GET

Contient les données envoyées via l'URL (méthode GET). Utile pour récupérer des paramètres dans l'URL. 

<?php
// URL : page.php?nom=Alice
echo "Bonjour " . htmlspecialchars($_GET['nom']);
?>

3. $_POST

Contient les données envoyées via un formulaire HTTP en méthode POST. 

<?php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $message = $_POST['message'] ?? '';
    echo "Message reçu : " . htmlspecialchars($message);
}
?>

4. $_SERVER

Fournit des informations sur le serveur et l'environnement d'exécution (ex. nom de script, adresse IP, méthode de requête). 

<?php
echo "Vous êtes sur la page : " . $_SERVER['PHP_SELF'];
echo "<br>Méthode HTTP : " . $_SERVER['REQUEST_METHOD'];
?>

5. $_SESSION

Permet de stocker des données persistantes entre plusieurs requêtes d'un même utilisateur (session). 

<?php
session_start();
$_SESSION['utilisateur'] = 'Alice';
echo "Utilisateur connecté : " . $_SESSION['utilisateur'];
?>

6. Autres superglobales

  • $_COOKIE : Données des cookies envoyés par le client
  • $_FILES : Informations sur les fichiers téléchargés
  • $_ENV : Variables d'environnement
  • $GLOBALS : Accès à toutes les variables globales

7. Sécurité

Toujours valider et nettoyer les données issues des superglobales avant de les utiliser pour éviter injections ou failles.

8. Résumé

  • Les superglobales sont accessibles partout dans le script
  • $_GET et $_POST récupèrent les données envoyées par formulaire
  • $_SERVER donne des infos sur la requête et le serveur
  • $_SESSION permet de gérer des données persistantes utilisateur
