Chapitre 61: CodeIgniter

1. Qu'est-ce que CodeIgniter ?

CodeIgniter est un framework PHP léger et rapide conçu pour les développeurs qui ont besoin d'un outil simple et élégant pour créer des applications web complètes. Il utilise une architecture MVC et vise à minimiser la configuration nécessaire tout en restant performant.

2. Pourquoi utiliser CodeIgniter ?

  • Léger : très peu de dépendances, rapide à déployer.
  • Facile à apprendre : idéal pour les débutants.
  • Documentation claire : bien structurée et facile à suivre.
  • Performance : excellent temps de chargement même sans cache complexe.
  • Souplesse : possibilité d'utiliser CodeIgniter comme micro-framework ou complet.

3. Prérequis

  • PHP ≥ 7.4 recommandé
  • Serveur web (Apache, Nginx...)
  • Composer (optionnel mais utile)

4. Installation de CodeIgniter

# Installation via Composer
composer create-project codeigniter4/appstarter mon-projet

# Accéder au dossier du projet
cd mon-projet

# Lancer le serveur de développement
php spark serve

5. Structure d’un projet CodeIgniter

  • app/ : contient les contrôleurs, modèles et vues.
  • public/ : dossier public accessible depuis le navigateur.
  • system/ : cœur du framework.
  • app/Config : fichiers de configuration.

6. Exemple de route simple

// app/Controllers/Accueil.php

namespace App\Controllers;

class Accueil extends BaseController
{
    public function index()
    {
        return 'Bonjour depuis CodeIgniter !';
    }
}

Et dans app/Config/Routes.php : 

$routes->get('/bonjour', 'Accueil::index');

7. Console CodeIgniter (Spark)

CodeIgniter 4 intègre une console CLI nommée spark pour créer des fichiers, lancer le serveur, générer des clés, etc. 

php spark list
php spark make:controller MonController

8. Conclusion

CodeIgniter est un excellent choix pour ceux qui cherchent un framework PHP rapide, léger et simple à prendre en main. Il permet de développer des applications web performantes sans complexité excessive.
