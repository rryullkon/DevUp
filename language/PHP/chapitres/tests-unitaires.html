Chapitre 62: Tests unitaires

1. Qu'est-ce que les tests unitaires ?

Les tests unitaires sont des tests automatisés visant à vérifier le bon fonctionnement des plus petites unités de code, généralement des fonctions ou méthodes individuelles, afin d'assurer leur fiabilité.

2. Pourquoi utiliser PHPUnit ?

  • Framework de tests unitaires le plus populaire en PHP.
  • Facilite l’écriture, l’exécution et la gestion des tests.
  • Intégration possible avec des outils CI/CD.
  • Documentation et communauté actives.

3. Installation de PHPUnit

# Installer via Composer (global ou local)
composer require --dev phpunit/phpunit

4. Exemple simple de test avec PHPUnit

// tests/CalculatorTest.php

use PHPUnit\Framework\TestCase;

class CalculatorTest extends TestCase
{
    public function testAddition()
    {
        $result = 2 + 3;
        $this->assertEquals(5, $result);
    }
}

5. Exécuter les tests

# Via Composer
vendor/bin/phpunit tests

# Ou globalement si phpunit est installé globalement
phpunit tests

6. Bonnes pratiques

  • Tester des cas simples et complexes.
  • Isoler les tests pour éviter les dépendances entre eux.
  • Nommer clairement les méthodes de test.
  • Intégrer les tests dans un workflow de développement continu.

7. Conclusion

L’intégration des tests unitaires avec PHPUnit est une étape clé pour améliorer la qualité et la robustesse de vos applications PHP. Cela permet de détecter rapidement les régressions et d’assurer la maintenabilité du code.
