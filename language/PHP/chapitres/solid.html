Chapitre 80: SOLID Principles

1. Introduction

Les principes SOLID sont un ensemble de bonnes pratiques en programmation orientée objet visant à améliorer la maintenabilité, la flexibilité et la compréhension du code. Ils ont été popularisés par Robert C. Martin (Uncle Bob).

2. Les 5 principes SOLID

  1. S - Single Responsibility Principle (SRP)
    Une classe doit avoir une seule responsabilité, une seule raison de changer.
  2. O - Open/Closed Principle (OCP)
    Les entités doivent être ouvertes à l’extension, mais fermées à la modification.
  3. L - Liskov Substitution Principle (LSP)
    Les objets d’une classe dérivée doivent pouvoir remplacer ceux de la classe de base sans altérer le comportement attendu.
  4. I - Interface Segregation Principle (ISP)
    Il vaut mieux plusieurs interfaces spécifiques que de grandes interfaces générales.
  5. D - Dependency Inversion Principle (DIP)
    Les modules de haut niveau ne doivent pas dépendre des modules de bas niveau, mais des abstractions.

3. Exemples simples

Single Responsibility Principle (SRP)

<?php
// Mauvais exemple : une classe qui fait trop de choses
class User {
    public function save() {
        // Code pour enregistrer l'utilisateur en base
    }

    public function sendEmail() {
        // Code pour envoyer un email à l'utilisateur
    }
}

// Bon exemple : séparer les responsabilités
class User {
    public function save() {
        // Enregistrer l'utilisateur
    }
}

class EmailSender {
    public function sendEmail(User $user) {
        // Envoyer un email
    }
}
?>

Open/Closed Principle (OCP)

<?php
interface Logger {
    public function log(string $message);
}

class FileLogger implements Logger {
    public function log(string $message) {
        // Log dans un fichier
    }
}

class Application {
    private Logger $logger;

    public function __construct(Logger $logger) {
        $this->logger = $logger;
    }

    public function run() {
        $this->logger->log("Application démarrée");
    }
}

// On peut changer facilement le type de logger sans modifier Application
$app = new Application(new FileLogger());
$app->run();
?>

4. Conclusion

Appliquer les principes SOLID aide à écrire un code plus robuste, évolutif et plus facile à maintenir. Ces principes sont essentiels pour tout développeur PHP souhaitant produire un code de qualité professionnelle.
